## 2. 데이터 시각화를 하는 이유

~~~
- 데이터 시각화의 정의가 모호한 면이 있음 : Information Design / Data Visualization / 
                                         Data, Information, Knowledge, Wisdom
- 많은 양의 데이터를 빠르고 정확하게 처리하는 것은 컴퓨터가 잘하지만, 패턴과 의미를 찾는 것은 사람이 잘하는 일이다. 즉, 데이터 시각화는 컴퓨터와 사람이 협력하는 방식 중 하나이다.
- 시각화는 그 자체로 인터랙티브한 것이다.
- 안 봐야하는 데이터를 안볼 수 있게 하는 것.
~~~

~~~

## 3. 로또 시각화

~~~
* 단점
- 가장 중요한 목적이 무엇인지 파악하기 힘들다.
- 결과데이터가 아닌 가공중인 데이터를 시각화 한 것 같다.
- 로또를 나타내는 상징적인 색상을 사용했으면 더 좋았을 것 같다.
- 강조하고 싶은 내용을 나타내는 색 선정이 잘못되었다. 배경이 어두울수록 반대색이 강조되는데, 빨강색으로 강조를 하려고 한것보다 흰색이 더 강조되어 보인다.
- 디테일을 강조하고 싶을 때 명도차이를 넣는데, 배경에 무늬를 넣은 것은 오류이다.
- 같은 내용을 나타내는 글자 크기가 달라서 인구수가 차이가 있어보인다.
- 시도별 당첨자수 비교 시각화에서 배경이 어두워서 특정 색을 강조하는 것처럼 보인다.

-> 중요한 내용은 자세히 보도록 한눈에 들어오게 시각화하지 않는 경우가 있다.
-> 디자인의 차이를 화이트, 레드, 크기 순으로 나타내었다.
-> 레인보우 컬러는 색생의 채도에 따라 강조되는 느낌이 있기 때문에 신중히 사용해야 한다.
-> 확률보다는 빈도가 더 와닿는 연구가 있다.
~~~
